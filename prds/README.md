# Product Milestone PRDs

This folder breaks the final product into milestone-sized PRDs instead of one oversized spec.

## Milestone Sequence
| Milestone | PRD | Why it matters |
|---|---|---|
| M1 | `PRD_01_Verification_Engine_Foundation.md` | Proves the local-first approach is accurate enough to trust and fast enough to demo |
| M2 | `PRD_02_Single_Label_Reviewer_Workbench.md` | Delivers the core single-label reviewer experience the take-home will be judged on |
| M3 | `PRD_03_Batch_Review_Beta.md` | Adds the most valuable differentiator once the happy path is stable |
| M4 | `PRD_04_Submission_and_Reviewer_Readiness.md` | Turns the prototype into a polished, evaluator-ready final product |

## Recommended Cut Line
- Must ship: M1, M2, and M4.
- Ship only if stable: M3.

## Why This Ordering
- `presearch.md` makes clear that a working core experience matters more than breadth.
- The existing golden-set evals already give us a measurable quality bar for the distilled-spirits MVP.
- Batch is important, but it should expand a stable core rather than compete with it.
